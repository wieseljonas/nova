# Sandbox & Code Execution

Aura has access to a sandboxed Linux VM powered by [E2B](https://e2b.dev). This allows her to run shell commands, execute code, process files, and interact with the filesystem safely ‚Äî without risking the production environment.

**Required env var:** `E2B_API_KEY` (and optionally `E2B_TEMPLATE_ID` for a custom template)

---

## `run_command`

Execute a shell command in the sandboxed Linux VM.

| Param | Type | Required | Description |
|-------|------|----------|-------------|
| `command` | string | **yes** | Shell command to run, e.g. `git clone https://github.com/org/repo.git` |
| `timeout_seconds` | number | no | Timeout in seconds (default 120, max 750). Use higher values for long-running agent commands. |
| `workdir` | string | no | Working directory, e.g. `/home/user/repo`. Defaults to `/home/user`. |

**Returns:** exit code, stdout, stderr.

## File system

The sandbox persists between messages **within a session** ‚Äî files and state are preserved across tool calls in the same conversation. The sandbox may auto-pause between sessions; check `/home/user/` to verify state is intact before assuming files exist.

Common paths:
- `/home/user/` ‚Äî working directory
- `/home/user/aura/` ‚Äî Aura's own codebase (cloned at startup)
- `/home/user/downloads/` ‚Äî files saved to disk via `download_slack_file` or `download_email_attachment`

---

## Isolation model

The sandbox is isolated from the Vercel runtime. It **cannot** access:
- The production PostgreSQL database (use `psql "$DATABASE_URL"` after extracting the URL via `skill-vercel-env-extraction`)
- Slack tokens (unless extracted and passed explicitly)
- Other production env vars (must be fetched from Vercel API using the `skill-vercel-env-extraction` playbook)

This isolation is intentional ‚Äî Aura can run untrusted code without risk to the production system.

## Timeout guidance

| Task | Recommended timeout |
|------|-------------------|
| Quick commands (`ls`, `grep`, `git log`) | 10‚Äì30s |
| Code execution, file processing | 60‚Äì120s |
| `claude` agent tasks (multi-file changes) | 400‚Äì750s |
| Long web crawls or batch processing | 300‚Äì600s |

The Vercel function timeout is 800s. Leave a 50s buffer for overhead ‚Äî use 750s maximum.
